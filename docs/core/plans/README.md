## Considered and rejected — do not re-report next audit

- *"Unauthenticated attackers can stream 25 MiB bodies because auth runs after body read"* — *false*: auth runs **before** `readBody` (`server.ts:210-216`).
- *Host-header trust when no API key is set* — documented two-mode design (`README.md` "Solve Authorization"); the code comment at `server.ts:496-497` shows it is deliberate.
- *Hard-coded 120s gmsh timeout should be client-configurable* — rejected; client-controlled timeouts worsen the DoS surface. Concurrency caps (plan 002) are the right lever.
- *Temp-dir leak on gmsh crash* — cleanup is in `finally` with `rm(..., { recursive: true, force: true })` (`gmsh.ts:120-122`); only a host-process crash leaks, which a container restart clears.
- *README documents proxy health fields (`containerBound`, `coreCloudAvailable`, `containerRunnerVersion`) that don't exist in this repo* — those fields are added by the OpenCAE app proxy in a different repo; plan 005 adds a clarifying sentence instead of deleting them.
- *`perLoad`/`loads` diagnostics aliasing is fragile* — already fixed in `292a6eb`; the comment at `loads.ts:125` documents the invariant. Style note only.
- *Per-frame nodal recovery is wasteful because results could be lazily evaluated* — recovery itself is required per frame (stresses differ per frame); the actual waste is per-frame **geometry** recomputation, which plan 004 fixes.
- *Viewer throws instead of returning error objects* — documented boundary behavior (`docs/validation/core.md`: invalid results "must be rejected by solver, cloud, or viewer boundaries").
- *Degenerate elements silently produce wrong dynamic masses (originally reported high-severity)* — downgraded, see Backlog: unreachable through production call graphs because stiffness assembly errors first.
- *`throw` vs result-object inconsistency across packages; `solver-cpu/results.ts` is 895 lines and should be split* — real debt, but high-risk refactors with low current payoff; revisit after plan 003's net exists and before any Tet10 work.
- *No coverage tooling / coverage % unknown* — deliberate non-finding; risk-weighted gaps are covered by plan 003 instead.
- *JSON depth-limit for request bodies* — bounded by the 25 MiB body cap; deep-nesting amplification is real but only reachable by authorized/platform-internal callers. Revisit if the service is ever exposed more directly.
- *(engineering pass)* *Unit-scale functions are brittle string matches* (`stressToMpaScale`/`lengthToMmScale` in `packages/solver-cpu/src/results.ts`) — the factors are **correct** for both declared systems (independently verified); "fragile if a third unit system is added" is hypothetical. Revisit only when a new system is actually proposed.
- *(engineering pass)* *"gmsh defaults to second-order elements for imported CAD"* — **false**: gmsh's default `Mesh.ElementOrder` is 1; uploaded CAD produces Tet4 today. The real, narrower exposure (user-supplied `.msh` containing Tet10, rejected late) is Step 4 of plan 009.
- *(engineering pass)* *Pre-mesh DOF estimation before running gmsh* — rejected as speculative/unreliable (node count is not predictable from bbox + size field across geometries); plan 009 Step 6 does a deterministic post-mesh fast-fail instead.
- *(engineering pass)* *STL watertightness pre-check via a geometry library* — rejected (new dependency, duplicate of gmsh's own check); plan 009 Step 5 maps gmsh's failure to a clear error instead.
- *(engineering pass)* *Auto-converting user units (density, gravity) based on heuristics* — rejected: silent conversion is worse than silent trust; plan 006 documents conventions and warns on implausibility instead.
- *(engineering pass)* *Frame timing drift over 500 frames* — checked: accumulation error ≪ the 1e-12 emission tolerance; `round(time, 9)` stamps are sound (`packages/solver-cpu/src/dynamic-mdof.ts:124-144`).
- *(engineering pass)* *Zero-yield-strength silently omits safety factor* — behavior is safe (no fake SF); the silence is fixed as a diagnostic in plan 008 Step 2 rather than reported as a defect.

## Verified correct — engineering pass (do not re-audit; evidence read at `292a6eb`)

Independently checked during the 2026-07-02 engineering-validity pass (advisor hand-derivations plus two subagents; agreement required):

- **Newmark average-acceleration integrator** (β=0.25, γ=0.5): coefficients a0–a5, effective matrix `(1+a1·β_R)K + (a0+a1·α_R)M`, predictor/corrector algebra, and initial acceleration `M⁻¹F(t₀)` at zero initial state — all match the textbook form (`packages/solver-cpu/src/dynamic-mdof.ts:258-312`).
- **Rayleigh damping application**: `C·v` computed as `α·M·v + β·(K·v)` with the full CSR stiffness (no diagonal shortcut) (`dynamic-mdof.ts:326-333`).
- **Von Mises formula** (`packages/solver-cpu/src/element.ts:84-94`) — standard form, Voigt ordering consistent.
- **CG failure honesty**: `cg-not-converged` and `singular-system` return `ok: false` and propagate; a non-converged displacement field can NOT reach a user as "computed" (`packages/solver-cpu/src/sparse.ts:163-205`, `solver.ts` propagation). (Reported residual trustworthiness is a separate item — plan 007.)
- **Result unit conversions**: m→mm ×1000 / Pa→MPa ×1e-6 (and identity for the mm-MPa system) are correct for both declared systems (`packages/solver-cpu/src/results.ts`).
- **Cloud mesh normalization**: mm→m coordinate scaling at parse with consistent `m-N-s-Pa` stamping (`services/opencae-core-cloud/src/mesh/gmsh.ts:188,269,286`); **inverted elements rejected at intake** (`coreModelFromMesh.ts`).
- **Validation-suite tolerances match the docs**: axial/pressure/gravity force-balance tight; cantilever 0.05×–25× as documented; tighter aluminum cantilever bounds (25–45 MPa / 0.35–0.75 mm) present (`packages/solver-cpu/tests/core-validation.test.ts`).
